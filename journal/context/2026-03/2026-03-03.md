## 5:49:12 PM CST - Context Capture

PRD-24 Autonomous PRD Mode — Research findings on autonomous loop viability.

The core autonomous loop design (/prd-start → implement → /clear → auto-resume → repeat) cannot work with current Claude Code capabilities. Two blockers discovered:

1. Claude cannot programmatically invoke /clear — it's a built-in CLI command, not a tool. The only auto-clear mechanism is ExitPlanMode (plan mode exit), which requires user menu selection.

2. SessionStart hook additionalContext is informational, not imperative. The prd-loop-continue.sh hook fires correctly on /clear and its stdout IS injected into Claude's context as a system-reminder. But Claude treats it as background information, not as a directive to execute. No amount of "MANDATORY ACTION REQUIRED" language changes this — the injection mechanism itself is passive.

What DOES work: YOLO skill descriptions with active trigger language ("INVOKE AUTOMATICALLY after...") DO drive proactive skill invocation within a single session. The /make-autonomous and /make-careful symlink-based mode switching works correctly. The SessionStart hook serves as a useful reminder when the user manually runs /clear.

Decision 10 added to PRD: autonomous loop descoped. Value of /make-autonomous reframed as within-session proactive invocation + frictionless permissions, not cross-session autonomous looping.
