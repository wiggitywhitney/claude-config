### Summary
The developer upgraded the `/verify` skill to emit structured error transcripts when verification phases fail. Instead of just showing raw error output, the `verify-phase.sh` script now captures command output and generates a JSON object containing the phase name, command, exit code, timestamp, and the last 20 lines of output whenever a phase fails. This JSON gets printed to stdout with a `VERIFY_ERROR_CONTEXT:` marker and also persisted to a temp file so the system can detect if the same phase keeps failing across restart cycles.

The SKILL.md documentation was updated to explain how to parse this JSON and produce a targeted error summary with a concrete suggested fix, rather than generic troubleshooting advice. The developer also explicitly scoped out Phase 4 (which uses `security-check.sh`) since that script doesn't emit the structured error marker. Thirteen new integration tests were added to validate the JSON emission, file persistence, and that passing commands don't leak error context into the output. All 412 tests in the suite pass.
