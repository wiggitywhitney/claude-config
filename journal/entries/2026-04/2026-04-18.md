### Summary
The cost tracker's aggregation logic was changed to key data by (session, model) pair instead of by session alone. Previously, when a session switched between different models mid-session, all tokens got priced using the first model's rates. Now the script tracks tokens separately for each model within a session, so each model's tokens are priced at their correct rates.

The fix involved restructuring the AWK script's data structures. Instead of storing values keyed by session ID (`sess_input[sid]`, `sess_model[sid]`, etc.), the developer switched to composite keys using session ID and model together (`pair_input[pair]`, `pair_model[pair]`). The loop that calculates costs now iterates over these (session, model) pairs instead of just sessions.

A regression test was added to catch this bug in the future. It creates a session that uses Claude Sonnet (pricing at $1.50 for 500k input tokens) and Claude Opus (pricing at $2.50 for the same amount), expecting a total of $4.00. The old code would have incorrectly yielded $3.00 by applying Sonnet's rate to all tokens.
