## 7:49:38 AM PDT - Commit: 03af967

### Summary
The developer added a PostToolUse hook that fires after PR merge commands to remind about cleaning up branches and confirming linked issues are closed, along with supporting documentation and tests.

The developer had noticed that merged PRs weren't getting their feature branches cleaned up locally and remotely. They identified three potential places to address this: in CLAUDE.md, in a PRD skill, or as an automated hook. They decided to go with a combination approach—adding a reminder to CLAUDE.md and building a hook that would fire whenever `gh pr merge` runs.

The developer created a new Bash script called `suggest-branch-cleanup.sh` that acts as a PostToolUse hook. It watches for Bash tool calls containing the `gh pr merge` command and, when detected, outputs an advisory message reminding to delete the feature branch locally and from the remote, and to confirm the linked GitHub issue was closed (either through an auto-close via a "Closes #NNN" line in the PR, or by closing it manually). The hook is purely advisory—it always exits successfully and doesn't block anything.

To ensure the hook works correctly, the developer wrote detailed tests covering the main cases: that the advisory fires on various forms of `gh pr merge` commands (basic merge, squash, rebase, and chained commands), that it stays silent for other GitHub CLI commands and git operations, that it ignores non-Bash tools entirely, and that it handles edge cases like invalid JSON and empty commands gracefully.

The developer updated the global CLAUDE.md file to add two new sections to the Git Workflow area: a direct instruction to delete feature branches after merging a PR, and a reference to GitHub CLI fork gotchas (which covers how `gh pr create` targets upstream by default). They also added references to two new gotcha documents for mmdc/mermaid-cli and social platform video uploads, and added a new section documenting eval run setup for the spinybacked-orbweaver-eval project.

The hook was registered in the settings.json configuration so it would actually run as part of the tool pipeline. The developer also documented the hook in hooks-reference.md alongside the other PostToolUse hooks, keeping the description brief but clear about what it does and when it fires.

### Summary
The developer fixed a false positive bug in a branch cleanup hook that was triggering on failure messages. The hook checks whether a GitHub pull request merge succeeded by looking for the word "merged" in the response, but this was matching error messages like "pull request was not merged" that contain the word but indicate failure.

The fix tightened the detection to look for the specific phrase "merged pull request" instead of just "merged". This prevents the hook from firing when a merge actually failed. The developer updated the check in the main script and added regression tests covering both the original success case and the new "was not merged" failure case, testing it across both the Python test suite and the Bash integration tests. The tests confirm the hook now stays silent when it encounters that particular failure message.
