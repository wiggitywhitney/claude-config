While `suggest-write-prompt` focuses on prompt quality during implementation handoffs, the new hook addresses completeness of planning context—ensuring that decisions, open questions, and enough detail to act on are captured when work moves from planning into a PRD or GitHub issue. It fires only at actual handoff moments: when `gh issue create` succeeds or when a new PRD file is created. When triggered, it asks three concrete questions about whether the document contains the necessary context for a future AI to work from. An 18-test suite covers the trigger cases and scenarios where the hook should stay quiet, like editing an existing PRD mid-session.

Both scripts were fixed by adding simpler patterns (`prds/*` and `rules/*`) alongside the absolute ones, with test cases added to verify the fix. The feature branch was then merged into main with all tests and documentation in place.
