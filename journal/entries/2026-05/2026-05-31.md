## 11:11:59 AM CDT - Commit: 3ba8b6c

### Summary
The developer added a pre-commit advisory check that fires when the prompt file is staged, asking three diagnostic questions without ever blocking the commit.

The new script, `check-prompt-generality.sh`, detects when `src/agent/prompt.ts` is staged and prints three questions to stderr: whether the guidance applies to any project (not eval-target-specific), whether it addresses root causes rather than symptoms, and whether examples use synthetic namespaces instead of real eval-target names. The hook always exits with status 0, making it purely advisory. It's silent when the prompt file isn't staged or when nothing is staged at all.

The developer integrated this into the pre-commit dispatcher by adding a call to the new check in `hooks/git/pre-commit`, aligning it with the existing checks for branch protection and progress tracking. The implementation includes a test suite covering the expected behavior: staying silent when nothing relevant is staged, printing the advisory when the prompt file is detected, verifying each of the three questions appears in the output, and confirming the exit code is always 0 even when staged alongside other files.

The work came together through test-driven development. The developer wrote failing tests first to specify the exact behavior needed—silent exits, specific question text, the advisory header—then implemented the minimum bash logic to make them pass. The tests verify edge cases like similarly named files (`prompt-v2.ts`) that shouldn't trigger the check, ensuring the path matching is precise. All tests pass and the hook integrates cleanly into the existing git workflow without disrupting commits.
