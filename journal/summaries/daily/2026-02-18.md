## Key Decisions

- **Scope Lint Phase to Changed Files**: Create a dedicated script to run linting only on files that have actually changed, with context-aware scoping for commit, push, and PR hooks.

- **Create detailed Testing Decision Guide**: Map five project types to concrete testing strategies with specific coverage targets and incorporate research from multiple sources.

- **Define Testing Rules for AI-Assisted Development**: Establish Always/Never patterns that separate deterministic from non-deterministic logic and prioritize testing security-critical and business logic code.

- **Add Anti-Suppression Instruction to Verification Hooks**: Explicitly forbid lint-disable comments and suppression annotations in hook deny messages, requiring Claude to fix underlying code problems instead.

- **Per-Language Rules Factored Out of CLAUDE.md**: Store language-specific rules in separate files to avoid burning context on rules for languages not in use.

- **Refactor CLAUDE.md Before Writing Templates**: Audit and refactor existing configuration using reference implementation patterns, then extract the patterns into templates rather than designing templates from theory.

- **Rename Research File to Neutral Language**: Remove personal name references from research analysis to make documentation more universally applicable.

- **Per-Repo Rule Overrides via Dotfiles**: Allow individual repositories to opt out of globally-enforced rules using dotfiles like `.skip-e2e` and `.skip-integration`.

- **Professional Commit Messages — No AI Attribution**: Commit messages must never reference Claude, AI, or include Co-Authored-By AI attribution; enforce via PreToolUse hook.

- **Same Org, Different Tags for Telemetry**: Use existing Datadog organization with service-level tagging rather than creating a separate org, allowing future migration if needed.

- **Full Block on ABOUTME Header Enforcement**: Implement PreToolUse hook that blocks Write and Edit operations without ABOUTME headers to ensure 100% adoption.

- **Language-Agnostic Header Support**: Support multiple programming languages from initial implementation with comment syntax mapping.

- **Add Milestone Task Tracking to PRD Skills**: Create tasks automatically for each unchecked milestone item, mapping tasks 1:1 with PRD milestone checkboxes.

- **Test Tier Enforcement Hooks with Dotfile Opt-Outs**: Enforce unit, integration, and end-to-end test presence via hooks with respect for `.skip-e2e` and `.skip-integration` opt-outs.

- **Research-First, Two-Tool Evaluation**: Research both CodeRabbit CLI and Codex before committing to integration, with hands-on testing against real code.

- **Lean PRD Workflow Approach**: Keep structured data capture skills but replace workflow routing with global CLAUDE.md rules and evaluate task recommendation capabilities.

- **Diff-Scoped npm Audit with Known Vulnerability Exclusion**: Only flag new vulnerabilities introduced by a branch, with `.audit-known-vulns` file support to exclude known issues.

- **Clarify Suppression Annotation Rules**: Explicitly allow `eslint-disable-line no-console` for intentional CLI output while maintaining strict stance against suppressing other errors.

- **Consolidate Project-Level CLAUDE.md Files**: Standardize behavioral instructions across repositories by moving files to `.claude/CLAUDE.md` and removing duplicated content.

- **Create Global Vals Usage Guide**: Provide shared reference documentation for secrets management tool usage across projects.

- **Milestone Reorder and Documentation Principles**: Group related hook work together, adopt execute-then-document principle for README, and expand Anki card creation to capture new knowledge from PRD work.
