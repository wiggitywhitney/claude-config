# Daily Summary — 2026-02-25

## Narrative

The developer had a productive day spanning two major projects and a handful of smaller fixes.

They started early by fixing a Ruff linting error in the branch protection tests, resolving git executable paths at module load time using `shutil.which()` instead of relying on partial path strings. This affected 26 subprocess calls across the test file. A follow-up commit added a noqa comment to suppress an optional TRY003 lint rule on the guard clause that raises an error if git isn't found.

The bulk of the day was spent on two substantial pieces of work. First, they completed the CodeRabbit CLI integration research and implementation (PRD #5). The research phase involved hands-on evaluation of the CodeRabbit CLI with different flag combinations, discovering that the `--prompt-only` mode designed for AI agents returned zero findings while `--plain` mode flagged issues—a critical insight about tool limitations. They also researched whether Codex could replace CodeRabbit and concluded it couldn't; Codex is a general-purpose agent with restrictive weekly review caps, while CodeRabbit's hourly caps suit frequent local reviews. The recommendation that emerged was a hybrid approach: add CodeRabbit CLI as an advisory pre-push hook (catching issues before PR creation), keep the existing GitHub PR review as the merge gate, and leave the MCP server integration untouched since its overhead is negligible.

The implementation followed, creating a standalone `coderabbit-review.sh` script that runs the CLI with graceful degradation—if the CLI isn't installed or times out, the hook silently continues. The pre-push hook was updated to call this script after security checks pass, capturing findings and embedding them in the hook's output so Claude can address issues locally. They added 30 new tests covering various scenarios and got all 248 tests passing. During end-to-end testing, a real CLI review identified three issues that were fixed, validating the implementation. The timeout was bumped from 90 seconds to 7 minutes to match realistic PR review wait times. After final testing, the PRD was marked complete and archived.

Second, they created PRD #19 for rewriting the seven PRD skills to operate in "YOLO mode"—removing excessive confirmation checkpoints that slow down workflow. The document outlines nine milestones covering archival of current skill versions and rewriting each skill (prd-create, prd-start, prd-next, prd-update-progress, prd-update-decisions, prd-done, prd-close) to make autonomous decisions rather than pausing for user confirmation at nearly every step. The design principle is to act rather than ask, batch information instead of presenting it piecemeal, and only pause for genuine ambiguity. Since all global skills are symlinks into this repository, rewriting the files here automatically updates every project that uses them. A follow-up commit addressed CodeRabbit review findings by clarifying loop termination conditions and expanding the risks table with mitigation strategies.

Late in the day, the developer fixed the CodeRabbit integration to handle platform differences and prevent false findings. The script now checks for `timeout` first, then falls back to `gtimeout` (from GNU coreutils on macOS), and can run without a timeout as a last resort. They also filtered out banner and status lines from CodeRabbit output that were being incorrectly treated as findings. Test assertions were updated to be more realistic and the skip-coderabbit tests were fixed by ensuring they set up `origin/main` so git operations don't fail. Documentation was unified to use 7 minutes for PR polling timing across the board.

## Key Decisions

- **Resolve git executable path at module load time using `shutil.which()`** — Fixes Ruff S607 linting error by finding the full path to git instead of relying on partial path strings; includes runtime check that raises an error if git isn't found on the system PATH.

- **Suppress Ruff TRY003 lint rule on RuntimeError guard clause** — CodeRabbit flagged the inline exception string as optional; added noqa comment to suppress without refactoring the logic.

- **Codex is complementary to CodeRabbit, not a replacement** — Codex's review feature is secondary to its general-purpose agent design with restrictive weekly caps (10-25/week); CodeRabbit's hourly caps suit frequent local reviews better. Hands-on testing skipped in favor of research findings.

- **Hybrid integration: CLI pre-push hook + PR review, keep MCP server** — CLI and PR reviews catch different issues; CLI cannot replace PR's cross-file analysis. Pre-push hook catches issues in ~30 seconds before PR creation, potentially eliminating one review round-trip. MCP server overhead is negligible (~$0.10/month) and tools provide real value for checking status and resolving comments programmatically.

- **Implement CodeRabbit CLI as advisory pre-push hook** — Runs after blocking security and test checks pass; findings surface in additionalContext but never block the push. Graceful degradation when CLI not installed, auth fails, or timeout occurs. Advisory chosen over blocking because AI reviews are non-deterministic and false positives would block legitimate pushes.

- **Always compare against default branch for CLI review base** — Use origin/main or origin/master, not upstream tracking ref, to ensure full feature branch diff is reviewed matching what PR review will see.

- **Skip CodeRabbit CLI review when .skip-coderabbit exists** — Provides escape hatch for edge cases without modifying hook code; consistent with existing skip patterns.

- **Create standalone coderabbit-review.sh script** — Follows existing pattern of modular verification scripts; reusable for other integration points beyond pre-push hook.

- **Increase CodeRabbit CLI review timeout from 90 seconds to 7 minutes** — CLI review completes faster than the PR review cycle (~5-7 minutes); any review finishing within 7 minutes represents net time savings compared to waiting for GitHub-based review.

- **Archive current PRD skill versions before rewriting for autonomous operation** — Archive as `SKILL.v1-careful.md` keeps careful version co-located with skill for easy reference; ignored by Claude Code which only loads `SKILL.md`. Allows restoration if YOLO rewrites cause issues. In-place archiving chosen over separate directory to avoid skill name collisions.

- **Rewrite PRD skills in-place rather than creating separate YOLO variants** — All global skills are symlinks into this repo, so rewriting active files updates all projects automatically. Avoids duplicate skill names; simplifies management by maintaining single canonical version per skill.

- **Design YOLO skills to act autonomously while preserving workflow rigor** — YOLO mode means fewer pauses, not less rigor. Skills should make their own decisions when sufficient context exists; only stop for genuinely ambiguous situations or decisions with major implications.

- **Implement skill-specific autonomy patterns for each PRD skill** — prd-create skips interview if feature description provided; prd-start removes readiness checklist display; prd-next skips confirmation gate; prd-update-progress applies updates based on evidence; prd-update-decisions auto-detects PRD; prd-done auto-fills PR information; prd-close proceeds without confirmation if closure reason provided.

- **Add platform-specific timeout fallback for CodeRabbit script** — Check for `timeout` first, then fall back to `gtimeout` (from GNU coreutils on macOS), and can run without a timeout as last resort to handle macOS compatibility.

- **Filter out banner and status lines from CodeRabbit output** — Check whether raw output contains actual review issue blocks (lines starting with `File:`, `Type:`, or `Comment:`) before surfacing findings to prevent clean runs from polluting PR review with noise.
