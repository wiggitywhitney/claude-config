## Key Decisions

- **Add colocated integration test file detection for Node.js/TypeScript projects** — Projects using colocated naming conventions (e.g., `*.integration.test.ts` next to source files) were falsely reported as missing integration tests. Added find-based detection for `*.integration.{test,spec}.*` file patterns with maxdepth 4, excluding node_modules and .git, as a fallback to existing directory and script-based checks.

- **Add colocated e2e test file detection for Node.js/TypeScript projects** — Added symmetric pattern to integration detection for consistency, using find-based detection for `*.e2e.{test,spec}.*` file patterns.

- **Parse --repo flag from merge command for cross-repo GitHub API calls** — Hook couldn't resolve repository when using `gh pr merge --repo owner/repo` from a different working directory. Extract `--repo OWNER/REPO` from the merge command and use it directly for GitHub API calls instead of relying on cwd-based repo resolution.

- **Parse cd path from chained commands to resolve .skip-coderabbit from correct project directory** — Hook was checking for `.skip-coderabbit` in the session cwd instead of the target repo when commands used `cd /path && gh pr merge`. Extract `cd /path` from command chains (supporting both `&&` and `;` separators) and use that directory for dotfile detection, prioritizing cd path over hook input cwd when valid.

- **Integrate CodeRabbit CLI as advisory pre-push review step** — Created a new bash script that runs CodeRabbit locally on every push before PR creation, with graceful fallbacks when the CLI isn't installed and timeout management. Script surfaces findings in pre-push hook output without blocking the push. Added `.skip-coderabbit` dotfile option to opt out when needed.
