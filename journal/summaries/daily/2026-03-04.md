# Daily Summary — 2026-03-04

## Narrative

The developer investigated PRD #3, which aimed to route Claude Code's native OpenTelemetry telemetry to a local Datadog Agent. They discovered that Claude Code emits 8 metrics and 5 event types (via the logs protocol), but hit a fundamental blocker: Datadog's managed settings—distributed via MDM as part of the developer's employer infrastructure—override all user-level OpenTelemetry configuration and forcibly redirect telemetry to an internal endpoint (`ai-devx-api.us1.prod.dog`). This made it impossible to route data to a local agent or personal Datadog org, rendering the PRD infeasible.

To confirm the issue, they installed the official Datadog MCP server (a remote HTTP endpoint with OAuth authentication) and verified the managed settings override. While investigating, they also fixed an unrelated issue in the `/make-autonomous` skill where `prds-get` lacked a YOLO variant, causing symlink confusion. Both the MCP server addition and the YOLO variant fix were merged to main via PR #27.

Before closing PRD #3, the developer reverted the OTEL environment variables from `~/.claude/settings.json`, updated the research documentation in the advocacy-skills repo with their findings, and created 5 Anki cards capturing key learnings about Claude Code's telemetry structure, the logs protocol gotcha, the managed settings blocker, the official MCP server, and the hardcoded service name. They then archived the PRD to `prds/done/` with status "Closed — No Longer Feasible," documented the closure reason, and updated the GitHub issue with a detailed explanation.

## Key Decisions

- **Abandon PRD #3 due to managed settings override.** Datadog's MDM-distributed managed settings have the highest precedence and override all user OTEL environment variables. Telemetry is locked to the internal endpoint and cannot be redirected to localhost or a personal org. This is a fundamental architectural constraint, not a configuration error. A dual-export approach via Prometheus would require `OTEL_METRICS_EXPORTER` to be overridable, which it is not.

- **Use the official Datadog MCP server (remote HTTP + OAuth).** The official server is a remote HTTP endpoint at `https://mcp.datadoghq.com/api/unstable/mcp-server/mcp` that authenticates via OAuth, not a local npm package. This replaces the community package `@winor30/mcp-server-datadog`, which was incorrect. The tradeoff is that a remote endpoint requires network access, but it's the only legitimate option.

- **Create a YOLO variant for prds-get.** Added `SKILL.v1-yolo.md` to enable proper symlink behavior in make-autonomous mode. The content is identical to the standard skill since prds-get requires no behavioral changes for YOLO mode.

- **Archive PRD #3 to prds/done/ with Closed status.** Moved the PRD from active to done, updated the status to "Closed — No Longer Feasible," added the closure date and reason, and closed the GitHub issue with a detailed explanation of the managed settings blocker.
