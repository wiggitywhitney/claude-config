## Key Decisions

- **Expand `.gitignore` to docs-only exemption for branch protection** — Allow commits directly to main/master when only modifying documentation files or `.gitignore`, preventing accidental commits of configuration files while permitting safe documentation updates.

- **Increase acceptance gate timeout from 300 to 1800 seconds** — Acceptance tests that make real API calls need a longer execution window to prevent premature timeout failures.

- **Add ABOUTME headers to hook scripts** — Enforce consistent documentation across all code files to clarify purpose and behavior at a glance.

- **Extract ABOUTME and vals-secrets rules into standalone documentation files** — Replace circular symlinks with actual rule files, enabling global configuration to reference detailed rules via imports without duplication.

- **Never mock locally installed tools** — Local tools should be tested against real binaries; mocking provides false confidence and masks compatibility issues.

- **Add cloud resource lifecycle management section** — Require teardown plans before provisioning cloud infrastructure to prevent orphaned resources.

- **Async acceptance gate tests via GitHub Actions CI** — Moving tests to CI allows the hook to return immediately with advisory context, reducing developer idle time waiting for test results before PR creation.

- **CI job timeout must be at least 45 minutes** — spinybacked-orbweaver's 10 LLM-calling tests take approximately 28 minutes sequentially, with individual tests sometimes spiking past 600 seconds due to API latency.

- **Run test files in parallel in CI** — Split test execution into fast and slow groups to parallelize, reducing overall CI runtime.

- **CI output capture is more reliable than local background tasks** — Local background task output was truncated or lost multiple times; CI logs are reliably captured and accessible.

- **Async CI trigger key in verify.json with sync fallback** — Add `acceptance_test_ci` to enable GitHub Actions workflow triggering, but keep `acceptance_test` as fallback for when the GitHub CLI is unavailable.

- **Parallel matrix strategy for spinybacked-orbweaver CI workflow** — Split test execution into core tests (fast) and coordinator tests (slow bottleneck) to run concurrently.

- **M6 scope reduced to commit-story-v2-eval and scaling-on-satisfaction** — cluster-whisperer's acceptance gate was never merged to main; telemetry-agent-spec-v3 was excluded per user request.

- **Automate `run-acceptance` label detection and application** — Detect acceptance gate tests by checking for either `.github/workflows/acceptance-gate.yml` or `acceptance_test` key in verify.json, and automatically apply the label when PRD workflow creates feature PRs.
