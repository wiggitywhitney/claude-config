## Key Decisions

- **Implement auto-rewrite loop for low-scoring cards without human approval**: Cards scoring below 9/15 are automatically rewritten once, targeting weakest dimensions, then re-scored. If still below 9, accepted with a threshold note. Bounded to one rewrite attempt to prevent infinite loops. Removes user control but eliminates decision fatigue and ensures all saved cards meet minimum quality.

- **Establish 3-dimension card quality scoring rubric**: Memory anchor clarity (1–5), future-self accessibility (1–5), and concept versus detail balance (1–5), totaling max 15 with 9/15 passing threshold. Scoring integrated into Phase 1 workflow before card generation.

- **Wire Story-First Framing into Phase 1 workflow as explicit step**: Developers outline narrative arc (why it exists, what it is, how it connects, what was surprising) before organizing concepts into card-ready format, preventing procedural skipping.

- **Make hierarchical tags mandatory on all cards**: Every card must have at least one hierarchical tag using Anki's `::` convention with prefixes like `project::`, `tech::`, `concept::`, `source::` to enable filtered study sessions without multiple decks.

- **Adopt write-directly pattern for section authoring in /write-docs**: Removes user confirmation gate between sections. Sections written directly to file; user interrupts if revision needed. Modeled after reference implementation to reduce friction.

- **Add enforcement language preventing command-skipping and environment give-up in /write-docs**: Commands are never skipped, missing tools are installed rather than handed back to user, and error cases must be documented alongside success paths. Addresses gap where agent previously gave up on missing environments.

- **Integrate /research skill at technology entry points in /write-docs**: Phase 1 runs `/research` when documentation target involves new-to-project or rapidly-changing technology. Phase 2 runs it when failures appear to be version drift. Prevents building on stale patterns.

- **Add Broken Docs Detection as Phase 2 in /write-docs**: Tests existing documentation before writing new content. Scans for existing docs, executes code examples, produces findings table with Pass/Fail/Skipped status, gates on user decision.

- **Adopt full-flags rule for command examples in /write-docs**: Always use long-form flags (e.g., `--filename` not `-f`) to be self-documenting for unfamiliar users.

- **Distinguish project-level and system-level dependency installation in /write-docs Step 5a**: Project-level dependencies (npm, pip, go modules) auto-install without user confirmation. System-level tools (brew, apt, curl-pipe-sh) require user confirmation first.

- **Run /write-prompt review after all skill changes complete, not partway through**: Ensures detailed coverage rather than leaving subsequent changes unreviewed.

- **Implement glossary index for /anki and /anki-yolo skills**: Persistent index tracks which technologies, APIs, frameworks, and coined terms already have Pattern 1 coverage. Phase 1 scans conversation for new terms and cross-references against index. Missing terms automatically queued as Pattern 1 cards without user deferral. - **Glossary cards are never deferred; always create them in the same session**: When Glossary Index Check finds missing terms, those Pattern 1 cards are automatically included in Phase 2 batch with no defer option. Defer adds friction with no benefit.

- **Image dimensions target 800px wide PNG format**: Community-tested sweet spot with no official Anki spec. PNG preferred over JPG for lossless quality and transparency. WebP ruled out due to inconsistent support across Mac and iOS clients.

- **Images must reside inside Obsidian vault at ~/Documents/Journal/anki/images/bank/**: ObsidianToAnki resolves `![[filename.png]]` via Obsidian's vault index. Images outside vault fail silently. Embed syntax needs no path prefix.

- **Image bank filenames must be globally unique using concept-name-bank.png convention**: Obsidian resolves by nearest filename match across vault. Duplicates create non-deterministic behavior. The -bank suffix namespaces files distinctly and prevents collisions.

- **Platform scope limited to macOS and AnkiMobile (iOS); no CSS workarounds needed**: Developer uses Anki desktop on macOS and AnkiMobile on iOS, not Android or AnkiDroid. CSS workarounds from research don't apply.
