# Daily Summary — 2026-04-05

## Narrative

The developer spent the day completing two major PRD initiatives and addressing code review feedback across multiple skills.

The first half of the day wrapped up PRD #48, which upgraded five skills with design patterns from mature Claude Code plugins. This involved implementing an image bank system for flashcard creation that persists visual associations across review sessions, maintaining a concept map that tracks which images go with which concepts, and auto-assigning images from a pool of art when users don't provide specific ones. The developer fixed a series of code review findings: adding Bash to the allowed-tools lists, correcting image selection logic from "oldest" to "first result," fixing shell script quoting, and clarifying the answer-reveal rule to allow logos and branded images as long as they don't spoil the answer on the card front. The PRD was then archived as complete.

After wrapping PRD #48, the developer created PRD #51 to add persistence to the `/research` skill. The core problem was that research runs produce valuable output that disappears when the conversation ends—there's no way to know what's already been researched or build incrementally on prior findings. The solution writes research to markdown files organized by topic under `docs/research/`, maintains a lightweight index to track what's been researched, and checks that index before starting new searches to avoid duplicate work. The developer worked through several design decisions with the AI, particularly around preventing lazy updates: they settled on requiring an explicit changelog entry for every modification that names what's being removed and added. They also decided against date prefixes in filenames for better discoverability and chose a simple one-line-per-file index format to keep things lightweight as the directory grows.

The developer then created PRD #52 to add prompt quality review to the prd-create skills. The insight was that PRD milestones function as de-facto instructions for future AI implementors, so they need prompt-quality review before being committed. This involved updating both the main and YOLO variants of the prd-create skill to include a `/write-prompt` review step after milestone definition. The developer also implemented a PostToolUse hook that suggests running `/write-prompt` when skill documentation files are edited, using a test-driven approach to write failing tests before implementing the hook itself.

For PRD #51, the developer implemented Phase 6 (now Phase 5 after renumbering) of the research skill to persist research output. The implementation uses `git rev-parse` to detect the repository root, converts research topics into filename slugs, and writes structured markdown files with date headers, update logs, findings, and sources. The developer decided to alert the user rather than fail gracefully if persistence fails, and to add Bash to the allowed-tools list to support git detection. They also implemented an update protocol that requires reading existing files in full before modification to prevent silent overwrites, with explicit changelog entries documenting what changed. The developer then added Phase 1 Step 5 to check the prior research index before conducting web searches, allowing the skill to surface existing findings and ask whether to extend, update, or accept them rather than duplicating effort.

Throughout the day, the developer engaged in thoughtful design discussions about avoiding AI laziness (particularly around research updates), preventing noise in advisory systems (deciding to surface skill-creator eval recommendations only on merge rather than every edit), and ensuring transparency in how research evolves across sessions.

## Key Decisions

- **Add Bash to allowed-tools in /anki and /anki-yolo skills** — The tag-cards.py step requires shell execution; Bash was missing from the allowed-tools list, blocking skill execution.

- **Replace "oldest" with deterministic first-result selection for image bank** — Glob sort order is not guaranteed to be oldest; deterministic selection uses explicit Glob listing and takes first result.

- **Image dimension specification: 800px on the longest side, no upscaling** — Images can be portrait or landscape orientation; don't upscale if already smaller than target size.

- **Every concept on a card gets an image** — Changed from selective assignment (glossary terms only) to universal assignment; removes the previous constraint limiting images to specific concept categories.

- **Image prompt offers three explicit choices** — "Yes, I'll provide one," "Pull from bank," and "Skip" replace the ambiguous Yes/No format; "Pull from bank" option hides when bank is empty.

- **Implement two-phase image bank workflow** — Phase 1 checks concept map for known vs. new concepts; Phase 2 auto-embeds mapped images or prompts user for new ones; oldest-first selection ensures deterministic, repeatable assignments.

- **Implement update protocol for existing research files** — Existing files must be read in full before modification; explicit changelog entry required naming specific removals and additions before rewrite; prevents silent overwrites.

- **Add git repository detection for research persistence** — `git rev-parse --show-toplevel` detects repo root; non-git context alerts user and accepts arbitrary file path as fallback.

- **Derive topic slug from research question for consistent filenames** — Topic converted to lowercase, hyphenated filename with no special characters; enables predictable file organization.

- **Create and maintain research index at docs/research/index.md** — Index tracks all research artifacts with file, description, and last-updated date; enables discovery and prevents duplicate research.

- **Write research file once after synthesis, revise via update protocol for Q&A** — Synthesis stands on its own merit; follow-up questions supplement rather than define content via update protocol.

- **PostToolUse hook architecture for advisory /write-prompt suggestions** — Advisory-only design fires on Write|Edit actions for SKILL.md and CLAUDE.md files; always exits cleanly, never blocks edits.

- **Broken docs must be fixed before proceeding** — Removed user bypass options in write-docs skill; broken documentation must be fixed before moving forward to writing new documentation.

- **Restrict auto-install to manifest-declared dependencies** — Auto-install only applies to packages already declared in project manifest or lockfile; requires user confirmation for anything not in the manifest.

- **Exclude /skill-creator eval from suggest-write-prompt hook** — Running eval on every file edit creates noise without proportional benefit; eval recommendations surface instead on milestone completion events in separate skills.

- **Surface /skill-creator eval recommendations only on merge** — Added Milestone 4 to PRD #52 to implement advisory in `/prd-done` that scans merged branch diff for changed skill files and recommends evaluation only when skills were actually modified.
