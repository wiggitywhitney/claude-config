## Key Decisions

- **Add PRD cross-referencing step after research file save** — After saving research, scan open PRDs semantically to identify milestones that would benefit from reading the research, then insert "Step 0: Read related research" links after each relevant milestone heading. Use semantic judgment rather than keyword matching, and add links to every relevant milestone independently since each gets a fresh AI instance. Use the Edit tool for targeted insertion rather than rewriting full files.

- **Implement duplicate link guard in PRD cross-referencing** — Check whether the line immediately after a milestone heading already contains a link to the same research slug before inserting, and skip insertion if a duplicate exists.

- **Ask user before assigning art pool images to new concepts** — In the anki-yolo workflow, prompt the user first with a list of new concepts and ask whether she has specific logos or images to provide. If she supplies images, save them with standardized dimensions. For concepts where she declines or doesn't provide images, fall back to deterministic assignment from remaining art pool images.

- **Document sips workaround for image processing with spaces in filenames** — Create a `resize_image` helper function that copies files to a temp path without spaces, processes them, and cleans up, since sips silently fails on files with spaces in paths despite proper quoting. Never upscale images; check dimensions first before calling sips.

- **Establish global location for technology gotcha rule files** — Gotcha rule files belong in `~/.claude/rules/` directory, never project-level rules/, since technology knowledge is cross-project and reusable. When an existing rule file exists for a technology, read it fully, add new findings, remove stale content, and update in place rather than creating duplicates.

- **Create global rule for PRD dependency management** — Every PRD milestone must be completable from main alone. Prevention at design time is cheaper than mid-implementation recovery. Recovery process: stop immediately, assess blocker type, pause blocked PRD, merge upstream PRD first, then resume. Never branch one PRD off another feature branch.

- **Establish safe rollout order for native git hook migration** — Install native hooks and verify in all repos before removing from settings.json, preventing coverage gaps if native hook installation fails in any repo.

- **Establish bats-core as standard for bash script testing** — Use bats-core (community fork) for all bash test suites. Document critical gotchas: `assert_output` and `assert_success` aren't built-in, `run` executes in a subshell, pipes require bash -c wrapper, GNU date shadows macOS date in PATH, and BSD grep rejects patterns starting with dash.
