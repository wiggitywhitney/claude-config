The system uses a dispatcher pattern where three main scripts—pre-commit, commit-msg, and pre-push—act as entry points that run check scripts from a `checks/` directory. Each dispatcher explicitly lists which checks it invokes, acting as a registry. All checks run even if one fails, so users see all errors at once rather than having to fix and retry. The pre-push dispatcher handles a tricky constraint by capturing stdin once at the start (since it can only be read once) and replaying it to each check script. If existing hooks are present, they're backed up with a timestamp before being replaced. The post-commit hook is explicitly preserved since other tools use it. ## Key Decisions

- **Replace CodeRabbit MCP tool with gh api commands** — The MCP tool was unreliable and incomplete. CodeRabbit posts findings across three distinct GitHub API channels, so missing any one means missing findings. Documented exact `gh api` commands with jq filters for consistent retrieval.

- **Dispatcher pattern with explicit check script registry** — Use a flat `checks/` directory with each dispatcher hard-coding which scripts it runs. Keeps structure simple and dispatchers human-readable, allowing future milestones to add scripts without changing dispatcher logic.

- **Run-all aggregation instead of fail-fast** — All checks run even if one fails, so users see all errors at once rather than having to fix and retry one at a time.

- **Pre-push stdin capture and replay pattern** — Capture stdin (refspec lines) once before any check script consumes it, then replay to each check script. Stdin can only be read once in a process, so this ensures all check scripts see the full refspec list.

- **Bootstrap script creates absolute symlinks with idempotency** — Create absolute symlinks (not relative) to dispatcher scripts. Idempotent detection of existing symlinks, with backup of existing real hook files before replacing. Explicitly preserve the post-commit hook since other tools use it.

- **Symlink resolution using readlink -f** — Resolve both the current symlink target and source path to absolute forms before comparing, with fallback to original readlink output if resolution fails. Ensures comparison works regardless of path representation.
