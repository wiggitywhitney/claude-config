## Key Decisions

- **Move ~/.claude/settings.json into version control via symlink** — Settings contain no secrets and are safe to commit. The symlink approach ensures writes from Claude Code's UI go directly to the repo file, keeping version control current automatically. Absolute paths in hook configurations work on machines with the same username and directory layout; new machines can patch paths during setup.

- **Create private claude-personal repo for memory files and per-project settings** — Memory files represent months of accumulated context and would be tedious to rebuild. Chat history is explicitly excluded due to unbounded growth and poor fit for git diffs. Per-project settings.local.json files should be captured to avoid re-approving tool permissions on new machines. Idempotent sync scripts are needed in both directions.

- **Create bootstrap script for new machine setup** — A single command should restore the full Claude development environment, including symlink setup, git hook installation, and memory/settings restoration. Auto-discovery of repos under ~/Documents/Repositories/ reduces maintenance burden. The script must be idempotent and handle partial setups gracefully.

- **Consolidate PRD and standalone issue fetching into a single GitHub API call** — Reduces API overhead and enables dependency analysis across both groups. Issue body content is needed for blocking relationship detection.

- **Extend suggest-write-prompt hook to fire on gh issue create** — The hook now handles two separate triggers: Write/Edit for file modifications and Bash for successful gh issue create commands. The advisory message clarifies that any AI-consumed document functions as a prompt.

- **Separate recursive and non-recursive rm commands in permission rules** — Plain rm for single files remains auto-approved. Recursive rm variants require confirmation. Ask rules take precedence over allow rules in Claude Code's permission resolution, enabling this approach.

- **Migrate four Claude Code PreToolUse hooks to native git checks** — Native hooks fire for every git operation, not just Claude Code's Bash tool, providing stronger enforcement. Commit-message.sh, branch-protection.sh, progress-md.sh, and test-tiers.sh replace the original hooks with additional features like docs-only exemptions and per-repo opt-outs.

- **Migrate pre-commit and pre-push hooks to native git checks with shared libraries** — Pre-commit-verify.sh runs build, typecheck, and lint checks. Pre-push-verify.sh runs security verification and CodeRabbit reviews, with escalated checks when pull requests are open. Shared library scripts handle project detection, verification phases, security scanning, and CodeRabbit integration.
