## Key Decisions

- **Rollout Scope Corrections for M4 Milestone**: `~/Documents/Journal` is a standalone git repo outside `~/Documents/Repositories/` and requires no special approval gate. `spinybacked-orbweaver` is closed/inactive and should be treated like any other repo without special approval gate. All repos including Journal and spinybacked-orbweaver can be installed in a single batch pass without per-repo confirmation.

- **Restructure M5 as a research-first investigation**: Instead of implementing code review plugin work in the native git hook migration PRD, spin it out into a dedicated PRD with a research spike at the beginning to decide whether to use the existing plugin or build a custom skill based on what the research uncovers.

- **Rephrase issue advisory message from instruction to confirmation**: Changed the git hook message from "Run /write-prompt on this issue body" to "Did you run /write-prompt on this issue body before creating it? If not, run it now." This acknowledges that Claude often completes the prompt before creating the issue, reducing redundant instructions while still catching cases where the prompt was skipped.

- **Establish explicit deferral protocol with user confirmation**: Silent deferrals disappear without memory in future sessions. Deferral is only appropriate for out-of-scope fixes (different repo, PRD-scoped work, merge conflict risk, important investigation needed). When deferral conditions apply, surface the decision to the user before proceeding and offer three explicit options: fix now, create a tracked GitHub issue, or let it go with awareness. Never proceed with a default choice silently. Tradeoff: adds friction to development flow but prevents issues from disappearing without user awareness.

- **Update prds-get skill to surface both PRDs and standalone issues consistently**: v1-yolo variant was stale and only surfaced PRD-labeled issues. Updated to match current behavior by fetching all open issues and splitting into PRDs and standalone issues, presenting both groups separately with active branch detection and dependency analysis.

- **Narrow Bash(rm *) allow rule to safe patterns**: Move general `rm *` to ask list and add targeted allow rules for common safe patterns: `*.tmp`, `*.log`, and `-f *.tmp`. Preserves frictionless workflow for legitimate temporary and log file cleanup while requiring confirmation for arbitrary deletions.

- **Parse git pre-push stdin payload for accurate ref detection**: Derive BRANCH and DIFF_BASE from actual refs being pushed rather than local checkout state. Handles edge cases: detached HEAD, non-origin remotes, multi-ref pushes. Falls back to existing behavior (@{upstream}, origin/main) when stdin is absent. Use REMOTE_NAME parameter for REVIEW_BASE instead of hardcoded 'origin'.

- **Map package manager to executor for fallback commands**: Add EXEC variable derived from detected package manager. Yarn classic uses 'yarn', Yarn Berry uses 'yarn dlx', npm/pnpm use 'npx'. Apply EXEC to fallback commands (tsc, eslint, jest, vitest) instead of hardcoded npx. Detect Yarn Berry presence via .yarnrc.yml file.
