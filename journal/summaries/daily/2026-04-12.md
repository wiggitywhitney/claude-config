## Key Decisions

- **Path prefix encoding must derive from $HOME, not whoami** — Claude Code encodes project directory paths by replacing forward slashes and dots with hyphens. On macOS systems where usernames contain dots (e.g., `whitney.lee`), using `whoami` returns the literal username but the actual project directory uses the encoded version. Computing the prefix by encoding `$HOME` through `sed 's|[/.]|-|g'` produces the correct prefix regardless of special characters.

- **Auto-discover git repos for hook installation with opt-out via dotfile** — The bootstrap script loops one level deep under `~/Documents/Repositories/` to find git repos, skips any with a `.skip-git-hooks` dotfile, and calls the hook installer for each one. Environment variables expose `REPOS_DIR` and `INSTALL_HOOKS_SCRIPT` for test isolation.

- **Use gh repo list filtered by activity window for repo sync** — The sync-repos script uses GitHub CLI to discover repos active within the last N months (configurable via `--months` flag, default 6), avoiding the need to maintain a static repo list.

- **git pull --ff-only for existing repos with warn-and-skip on failure** — Fast-forward-only pulls prevent silent merge commits. Repos with local changes or diverged history are surfaced as warnings, not errors.

- **Non-interactive --months flag instead of interactive prompt** — Keeps sync-repos consistent with bootstrap's flag style and makes it scriptable for pipelines.

- **sync-repos.sh is Milestone 5 of PRD #63, not a separate issue** — The script fits the "bootstrap a new machine" problem and is a prerequisite to bootstrap.sh, so it belongs in the same delivery.

- **Handle clone failures with explicit error reporting** — Clone operations that fail now report an `[ERROR]` message to stderr instead of silently reporting success.
