The developer spent the day working across two major features: completing the new-machine bootstrap system and evaluating a code-review plugin.

They also replaced GNU-specific date commands with portable Python one-liners for macOS compatibility and documented coreutils as a prerequisite. Once all fixes were in, they archived the completed PRD.

In the afternoon, the developer pivoted to evaluating the official code-review plugin from the Claude registry. They tested it against a real PR and found it caught different issues than CodeRabbit — the plugin identifies forward-compatibility gaps, convention inconsistencies, and historical context issues, while CodeRabbit catches security and correctness problems. This meant the tools were genuinely complementary, not redundant. The developer decided to integrate the plugin as a permanent part of the PR workflow, running it on every PR alongside CodeRabbit rather than treating it as a rate-limit fallback. They documented this decision in a research spike report, updated the project settings to enable the plugin globally, and revised the related PRD to expand documentation scope and eliminate the now-unnecessary task of building a custom skill.

## Key Decisions

- **Add external reader accessibility guidelines to progress documentation skill** — Entries were drifting toward internal references meaningless outside the immediate project context; guidelines shift focus from files changed to capabilities gained, with explicit examples of good versus bad formats.

- **Use hardcoded defaults plus per-repo opt-in for private file sync** — Hardcoded list (`journal/`, `.claude/design-decisions.md`) handles the common case with zero configuration; optional `.private-sync` file in each repo allows exceptions without changing core logic.

- **Implement backup as a standalone script separate from bootstrap** — Runs on the current machine (outbound) while bootstrap runs on the new machine (inbound); different phases warrant separate scripts, and this enables periodic backups beyond just pre-migration.

- **Reject `.private-sync` paths that escape the repo root via pattern matching and symlink resolution** — Paths could escape using relative traversal (`../../.git/config`) or symlinks; pattern check rejects absolute paths and traversal attempts, and symlink resolution verifies the resolved path stays within the repo root.

- **Fail fast if `claude-personal` has pre-existing staged changes before committing backup** — Previous implementation could silently include unrelated work in the same commit; now checks for pre-existing staged changes and exits with error if found.

- **Propagate non-zero exit code when git hook installation fails** — Hook failures were silently swallowed; now tracked and cause the script to exit 1 if any repo failed.

- **Replace GNU-only `date -d` with portable Python one-liners for macOS compatibility** — Tests used GNU-specific date arithmetic unavailable in macOS's BSD date; replaced with Python3 one-liners.

- **Document GNU coreutils as a prerequisite for sync-repos.sh** — The sync script uses GNU date for date arithmetic; added to prerequisites with installation instructions.

- **Run `/code-review` plugin on every PR as permanent complement, not just rate-limit fallback** — CodeRabbit and the plugin find different issue classes (security/correctness vs. forward-compatibility/conventions); running both provides genuine complementary coverage rather than redundancy.

- **Use Code Review Plugin as-is rather than building a custom skill** — Plugin tested against real PR and found genuine issues; already installed globally and demonstrably works; building a custom skill would duplicate existing work.
