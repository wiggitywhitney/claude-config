## Key Decisions

- **PRD #84 uses PRD file itself as execution state file** — Eliminates need for separate `_execution-state.md`; preserves atomic-commit invariant already built into workflow; disk-persisted and survives compaction natively.

- **Shell-driven Ralph loop architecture with fresh `claude -p` child sessions per milestone** — Avoids building parent Claude Code session (which has context limits and compaction risk); used existing skill infrastructure; fresh context per milestone caps blast radius of failures.

- **Single pause-marker mechanism (`**Paused — <reason>:** <detail>`) written into active milestone** — Consolidates stuck detection and human-in-the-loop handoff; consistent with treating PRD as state file; requires global rule documenting pause-handling protocol.

- **Extend `/make-autonomous` allowlist rather than redesign permissions** — Skill already covers ~85% of needs; additions cover TaskCreate, Agent, test commands via Bash, and ScheduleWakeup for headless child sessions.

- **Modify git pre-push hook to run tests unconditionally** — Closes gap where broken tests could accumulate before CI catches them; distinguishes unit tests (run on every push) from acceptance/e2e tests (which should not run per `verify.json` convention).

- **Bundle Ralph loop detection from PRD 58 into PRD 84 as M4** — Eliminates cross-PRD dependency; keeps terminology in the PRD that uses it correctly; detection and marker-writing are two halves of one mechanism.

- **Write active-run marker file (`.claude/autonomous-run.local`) in M3 orchestrator** — Contains PID, ISO8601 timestamp, and PRD number; removed on exit via SIGINT/SIGTERM traps and clean completion; enables M4 SessionStart hook to detect collisions.

- **Reference `/cost-tracker` in PRD 84 M6 and M7** — Cost visibility is essential for safely trusting unattended execution; `/cost-tracker` kept separate from PRD 84 (not bundled) because it has independent scope and loose coupling.

- **Cost-tracker aggregates by (session, model) pair instead of session alone** — Ensures tokens are priced at correct rates when a session switches models mid-session.

- **Add "Defer" disposition to code-review triage rubric** — Distinguishes findings that are real and worth addressing but too complex for inline fixes; deferred findings tracked in dedicated GitHub issues, not left untracked.
