This hook fires only at actual handoff moments: when a GitHub issue is created or a new PRD file is written. During development, both hooks had a glob pattern bug that missed relative paths like `prds/93-feature.md`; the developer fixed this by adding simpler patterns alongside the absolute ones and added test cases to verify the fix. A 22-test suite covers the expanded `suggest-write-prompt` hook, and an 18-test suite covers the new `suggest-planning-handoff` hook.

## Key Decisions

- **Drop `/skill-creator eval` advisory from `/prd-done` entirely.** The eval process is too expensive (300K tokens, several minutes per run) to recommend for routine skill changes. If it's ever surfaced in the future, it should guide users to decide whether it's worth running rather than recommending it unconditionally.

- **Extend `suggest-write-prompt` hook to fire for PRD files, rules files, and non-standard prompt filenames.** The hook was missing file types agents interact with regularly.

- **Create a separate `suggest-planning-handoff` hook instead of mixing into `suggest-write-prompt`.** Planning handoff (completeness of context) and implementation handoff (prompt quality) are different concerns. The new hook fires only at actual handoff moments (GitHub issue creation or new PRD file creation) and asks three concrete questions about whether the document contains necessary context.

- **Fire `suggest-planning-handoff` hook only on new file creation, not edits.** Using the Write tool (creates) versus Edit tool (modifies) distinction prevents repeated reminders during mid-session refinement.

- **Register `suggest-planning-handoff` hook in both Bash and Write PostToolUse matchers.** The hook needs to catch both `gh issue create` (via Bash) and new PRD file creation (via Write).

- **Fix path matching in both hooks to handle relative paths.** Glob patterns like `*/prds/*` missed relative paths like `prds/93-feature.md`. Added simpler patterns (`prds/*`, `rules/*`) alongside absolute ones.
