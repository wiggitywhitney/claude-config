# Weekly Summary — 2026-W10

## Week in Review

The developer spent the week building out an autonomous PRD workflow system and acceptance testing infrastructure. The work fell into two major threads that evolved as technical constraints became clearer.

Early in the week, they overhauled the PRD skills to operate autonomously—removing confirmation gates, auto-detecting target PRDs, and chaining skill invocations automatically. They built a SessionStart hook to bridge context gaps when developers run `/clear` between tasks, enabling continuous loops that re-read the PRD and pick up where they left off. They also created `/anki-yolo` for autonomous card generation without approval workflows. But midway through day 2, they discovered that Claude Code's skill override system is fundamentally broken—global skills don't reliably get shadowed by project-level ones. Rather than fighting the bug, they pivoted to a symlink-based system where `/make-autonomous` creates per-project symlinks to YOLO variants instead of copying files around. This eliminates the picker duplication entirely. They also discovered that the fully autonomous cross-session loop can't work because Claude can't invoke `/clear` programmatically, so they descoped the autonomous looping and reframed what `/make-autonomous` actually delivers: proactive skill invocation within a session and frictionless permissions.

In parallel, they built acceptance gate testing—a system for running expensive tests that make real API calls only after cheaper verification passes. They started by documenting the feature and implementing it in the pre-PR hook with a two-tier detection strategy (preferring `.claude/verify.json` with fallback to file patterns). Tests are advisory, never blocking, but require mandatory human review before proceeding. They rolled this out to four repositories with Anthropic API integrations. By day 5, they realized acceptance tests were blocking PR creation for up to 28 minutes with poor diagnostics, so they moved tests to GitHub Actions on day 6. The hook now triggers CI asynchronously and returns immediately, letting developers create PRs while tests run in the background. They set the CI timeout to 45 minutes based on real operational data from spinybacked-orbweaver showing tests can spike past 600 seconds due to API latency.

Throughout the week, they also added progress log automation—embedding PROGRESS.md updates directly into `/prd-update-progress` so logs stay in sync automatically. They added a pre-commit hook to enforce PROGRESS.md updates when PRD checkboxes change. They created an ABOUTME header enforcement system to document code file purpose. They automated the `run-acceptance` label strategy so expensive tests only run on feature PRs aligned with PRDs, not quick-fix PRs. And they closed PRD #3 after discovering that Datadog's managed settings—distributed via MDM as part of the employer infrastructure—override all user-level telemetry configuration, making it impossible to route Claude Code's native OpenTelemetry data to a local agent.

## Highlights

- **Redesigned PRD skill architecture from file-swapping to symlinks** — Eliminated Claude Code's skill override bug by removing global PRD skills entirely and having projects create symlinks to either YOLO or careful variants instead. Projects without `/make-autonomous` have no PRD skills in their picker, eliminating duplication.

- **Moved acceptance gate tests to GitHub Actions CI** — Tests that were blocking PR creation for up to 28 minutes now run asynchronously in CI with a 45-minute timeout. The hook returns immediately with advisory context, letting developers create PRs while tests run in the background.

- **Automated progress logging into PRD workflows** — Embedded PROGRESS.md updates directly into `/prd-update-progress` so logs stay in sync automatically. Added a pre-commit hook to enforce updates when PRD checkboxes change, eliminating manual log maintenance.

- **Built acceptance gate testing system** — Implemented a two-tier detection strategy for expensive API-based tests, making them advisory (never blocking) but requiring mandatory human review. Rolled out to four repositories with Anthropic API integrations.

- **Closed PRD #3 after discovering managed settings blocker** — Datadog's MDM-distributed managed settings override all user-level telemetry configuration, making it impossible to route Claude Code's native OpenTelemetry data anywhere but the internal endpoint.

## Patterns

- **Technical constraints driving architecture redesign** — The skill override bug on day 2 and the `/clear` invocation limitation forced rethinking the autonomous PRD system midway through. Rather than fighting the constraints, the developer pivoted to workarounds that fit within what Claude Code actually supports.

- **Operational data informing infrastructure decisions** — Real usage in spinybacked-orbweaver revealed that acceptance tests take 28 minutes and spike past 600 seconds, which directly informed the 45-minute CI timeout and the decision to move tests async.

- **Automation replacing manual enforcement** — Progress logs, PROGRESS.md updates, ABOUTME headers, and `run-acceptance` labels all shifted from "remember to do this" to "the system enforces it automatically," reducing cognitive load and preventing drift.

- **Documentation-first design for discovery** — The developer led with the "why" (tests that cost real money) and inline examples when introducing new features, recognizing that developers in other repos wouldn't know acceptance gates exist unless the documentation was clear and accessible.
