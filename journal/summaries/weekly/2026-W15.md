# Weekly Summary — 2026-W15

## Week in Review

The developer had a productive week spanning infrastructure work, feature completion, and tooling improvements across multiple projects.

Early in the week, they wrapped up the research skill persistence feature by adding cross-referencing between saved research files and project requirements documents. When research gets saved, the system now scans open PRDs to find relevant milestones and inserts "Step 0: Read related research" links, with duplicate guards to prevent stacking. They also discovered that macOS's `sips` tool silently fails on filenames with spaces and built a workaround that copies files to a temp path for processing. This led to broader work standardizing image handling across multiple skills and establishing a pattern that technology gotchas should live in global rule files rather than project directories, since that knowledge is reusable across repositories.

Mid-week, the developer shifted focus to infrastructure. They replaced an unreliable CodeRabbit MCP tool with direct GitHub API calls and discovered the tool was only checking one of three API channels where CodeRabbit posts reviews. They also moved the global Claude settings file into version control via symlink and planned a two-part system for migrating Claude's environment to new machines—one PRD for backing up memory files and per-project settings, another for a bootstrap script that orchestrates full environment restore. The bulk of the week went to migrating Claude Code hooks into native git checks. They built out a dispatcher pattern where three main scripts act as entry points that run check scripts from a `checks/` directory, with all checks running even if one fails so users see all errors at once. This involved creating four new verification scripts, migrating two more existing hooks, and adding six shared library scripts to support the system. By mid-week they'd completed the rollout across 25 repositories plus the Journal directory.

Late in the week, the developer addressed multiple rounds of code review feedback, fixing security issues in test-tier detection, improving shell script compatibility, and narrowing overly broad permission rules. They also worked on PRD verification automation—adding a pre-PR verification step that maps each success criterion to observable evidence in the codebase using a three-level check (Exists, Substantive, Wired). They implemented a two-layer system for syncing decision logs across open PRDs using both a global rule and a PostToolUse hook that fires when decision logs are updated. By week's end, they'd completed the claude-personal backup system for memory files and made substantial progress on the bootstrap-new-machine project, building a multi-step script that restores settings, memory files, git hooks, and per-project configurations. They also implemented a companion sync-repos script that uses GitHub CLI to discover and sync repositories across machines.
