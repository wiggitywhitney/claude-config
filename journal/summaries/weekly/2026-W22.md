# Weekly Summary — 2026-W22

## Week in Review

The developer built a new pre-commit advisory hook that runs whenever `src/agent/prompt.ts` gets staged. The hook fires three diagnostic questions to stderr—asking whether the guidance applies broadly, whether it addresses root causes, and whether examples use synthetic namespaces—but never blocks a commit since it always exits successfully. It's meant to be informational, nudging the developer to think about prompt quality without getting in the way.

The approach was test-first: nine failing tests specified the exact behavior upfront, then the bash implementation in `check-prompt-generality.sh` was written to make them pass. The tests cover the main cases (staying silent when nothing relevant is staged, printing all three questions when the prompt file is detected, always exiting with 0) and edge cases (similarly named files that shouldn't trigger the check). The new script integrated cleanly into the existing pre-commit dispatcher alongside branch protection and progress tracking checks, and PROGRESS.md was updated to document the work.
